## 简介

Modify-Cli 是一个命令行工具，用于修改文件。它提供了创建新项目的功能，支持 Vue 和 React 技术栈，并且可以选择是否使用 TypeScript。本指南将详细介绍如何在不同操作系统上安装和配置 modify-cli 工具。

## 安装前提条件

### Node.js 版本要求

在安装 modify-cli 之前，必须确保系统已经正确安装了 Node.js 环境：

- **最低版本要求**: Node.js 14.x 或更高版本
- **推荐版本**: Node.js 18.x LTS 或最新稳定版本
- **npm 版本**: npm 6.x 或更高版本

### 检查现有环境

在开始安装之前，请先检查您的 Node.js 和 npm 环境：

```bash
node --version
npm --version
```

如果未安装 Node.js 或版本过低，请访问 [Node.js 官方网站](https://nodejs.org/) 下载并安装合适的版本。

## 全局安装步骤

### 基本安装命令

modify-cli 支持通过 npm 进行全局安装：

```bash
npm install -g modify-cli
```

### 安装过程解析

当执行上述命令时，npm 将完成以下操作：

1. **下载包**: 从 npm 注册表下载 modify-cli 包
2. **解析依赖**: 安装所有必要的依赖包（chalk、commander、figlet、fs-extra、git-clone、inquirer、ora）
3. **配置 bin 字段**: 根据 package.json 中的 bin 字段设置可执行文件
4. **注册到 PATH**: 将 modify-cli 命令添加到系统的可执行路径中

## 安装过程详解

### package.json 中的 bin 字段配置

modify-cli 的 package.json 文件中定义了 bin 字段：

```json
{
  "bin": "/bin/index.js"
}
```

这个配置的作用是：
- **映射关系**: 将 `/bin/index.js` 映射为可执行命令 `modify-cli`
- **入口点**: 指定程序的入口文件为 `bin/index.js`
- **shebang 处理**: 文件顶部的 `#!/usr/bin/env node` 告诉系统使用 Node.js 来执行该脚本

### Node.js 如何处理 bin 字段

当 npm 安装 modify-cli 时，它会：
1. **复制文件**: 将 `bin/index.js` 复制到 npm 的全局 bin 目录
2. **设置权限**: 确保文件具有可执行权限
3. **PATH 注册**: 将 npm 的 bin 目录添加到系统的 PATH 环境变量中
4. **符号链接**: 创建指向实际文件的符号链接

### 依赖包说明

modify-cli 使用了多个关键依赖包：

- **chalk**: 用于创建彩色输出
- **commander**: 提供命令行界面功能
- **figlet**: 生成 ASCII 艺术字
- **fs-extra**: 增强的文件系统操作
- **git-clone**: 克隆 Git 仓库
- **inquirer**: 交互式用户界面
- **ora**: 进度指示器

## 验证安装

### 基本验证方法

安装完成后，可以通过以下方式验证 modify-cli 是否正确安装：

#### 方法一：查看帮助信息

```bash
modify-cli --help
```

正确的输出应该显示 modify-cli 的基本用法和可用命令：

```
Usage: modify-cli <command> [options]

Commands:
  create <project-name>  create a new project

Options:
  -V, --version          output the version number
  -h, --help             display help for command
```

#### 方法二：查看版本号

```bash
modify-cli --version
```

输出应该显示当前安装的版本号，例如：

```
v1.0.1
```

#### 方法三：测试基本功能

尝试运行一个简单的命令来测试功能：

```bash
modify-cli create test-project
```

如果安装成功，应该会启动交互式界面，允许您选择技术栈和配置选项。

### 高级验证

#### 检查 PATH 环境变量

验证 modify-cli 是否被正确添加到 PATH：

```bash
which modify-cli    # Linux/macOS
where modify-cli    # Windows
```

#### 检查可执行文件

直接检查 modify-cli 的可执行文件：

```bash
ls -la $(npm bin -g)/modify-cli
```

## 常见问题及解决方案

### 权限错误

#### 问题描述
在全局安装时遇到权限错误：

```
EACCES: permission denied, mkdir '/usr/local/lib/node_modules'
```

#### 解决方案

**方法一：使用 sudo（Linux/macOS）**

```bash
sudo npm install -g modify-cli
```

**方法二：更改 npm 全局目录权限**

1. 创建一个新的全局 npm 目录：
```bash
mkdir ~/.npm-global
```

2. 配置 npm 使用新的全局目录：
```bash
npm config set prefix '~/.npm-global'
```

3. 更新 PATH 环境变量：
```bash
export PATH=~/.npm-global/bin:$PATH
```

4. 重新安装：
```bash
npm install -g modify-cli
```

**方法三：使用 nvm 管理 Node.js 版本**

使用 nvm（Node Version Manager）可以避免权限问题：

```bash
nvm install --lts
nvm use --lts
npm install -g modify-cli
```

### 网络代理问题

#### 问题描述
在中国大陆地区，由于网络限制可能导致 npm 安装失败：

```
ERR_SOCKET_TIMEOUT
```

#### 解决方案

**方法一：使用淘宝镜像源**

```bash
npm install -g modify-cli --registry=https://registry.npmmirror.com
```

**方法二：配置 npm 使用镜像源**

```bash
npm config set registry https://registry.npmmirror.com
npm install -g modify-cli
```

**方法三：使用 yarn**

```bash
yarn global add modify-cli
```

### 磁盘空间不足

#### 问题描述
安装过程中出现磁盘空间不足错误：

```
ENOSPC: no space left on device
```

#### 解决方案

清理 npm 缓存：

```bash
npm cache clean --force
```

清理临时文件：

```bash
rm -rf ~/.npm/_cacache
```

或者使用 yarn 清理缓存：

```bash
yarn cache clean
```

### 版本冲突

#### 问题描述
系统中存在多个 Node.js 版本导致冲突：

```
Error: Cannot resolve module 'xxx'
```

#### 解决方案

使用 nvm 管理 Node.js 版本：

```bash
nvm install 18
nvm use 18
npm install -g modify-cli
```

## 操作系统特定配置

### Windows 系统

#### PowerShell 配置

1. **启用执行策略**：
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

2. **添加 npm 全局目录到 PATH**：
```powershell
$env:PATH += ";C:\Users\<用户名>\AppData\Roaming\npm"
```

3. **永久配置**：
通过系统属性 -> 高级 -> 环境变量，添加 npm 全局目录到 PATH。

#### CMD 配置

1. **检查 npm 全局目录**：
```cmd
npm config get prefix
```

2. **手动添加到 PATH**：
```
SET PATH=%PATH%;C:\Users\<用户名>\AppData\Roaming\npm
```

### macOS 系统

#### zsh 用户配置

1. **编辑配置文件**：
```bash
nano ~/.zshrc
```

2. **添加 npm 路径**：
```bash
export PATH=~/.npm-global/bin:$PATH
```

3. **应用更改**：
```bash
source ~/.zshrc
```

#### bash 用户配置

1. **编辑配置文件**：
```bash
nano ~/.bash_profile
```

2. **添加 npm 路径**：
```bash
export PATH=~/.npm-global/bin:$PATH
```

3. **应用更改**：
```bash
source ~/.bash_profile
```

### Linux 系统

#### Ubuntu/Debian

1. **更新 apt 缓存**：
```bash
sudo apt update
```

2. **安装 Node.js**：
```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

3. **配置 npm 全局目录**：
```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo "export PATH=~/.npm-global/bin:$PATH" >> ~/.bashrc
source ~/.bashrc
```

#### CentOS/RHEL/Fedora

1. **安装 Node.js**：
```bash
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

2. **配置 npm 全局目录**：
```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo "export PATH=~/.npm-global/bin:$PATH" >> ~/.bashrc
source ~/.bashrc
```

### 常见错误代码

#### ERR_DLOPEN_FAILED

**错误信息**：
```
Error: Cannot open shared object file: No such file or directory
```

**解决方案**：
1. 重新安装 Node.js
2. 清理 npm 缓存
3. 重新安装 modify-cli

#### ENOENT

**错误信息**：
```
ENOENT: no such file or directory
```

**解决方案**：
1. 检查 npm 全局目录权限
2. 重新配置 npm prefix
3. 重新安装 modify-cli

#### EACCES

**错误信息**：
```
EACCES: permission denied
```

**解决方案**：
参考前面的权限错误解决方案。
